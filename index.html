Welcome to the DMX information page

DMX is a framework which facilitates efficient development of domain-specific simulation modelling languages and tools. It is developed as a set of Eclipse plugins running under Eclipse Version 4.3 Kepler.

The following Eclipse plugins are required by DMX:

  • Eclipse Modeling Framework (EMF),
  • Xtend Version 2.4 or higher, available from here,
  • and Textual Editing Framework (TEF), available from this Eclipse Update Site.

DMX is an academic project. We try to have a stable running version in the latest commit. However, there might be some things not working from time to time. The purpose is to show the principle working of the framework.

In order to use DMX, you should import its plugins into your own Eclipse workspace. The contents provided at GitHub are listed below.

Contents

  • The folder dev-plugins contains the framework as a set of Eclipse plugins. These plugins should be imported into your Eclipse workspace.
  • The folder runtime-plugins contains an example project with a lot of DMX models. The project should be imported into the workspace of a second running Eclipse initiated from the first one.
    • The State Machine Language (SML) used in a paper which was presented at the SDL Forum 2013 (June 26-28) is available here.
